# Digital Identity Protection System - Architecture Documentation\n\n## System Overview\n\nA comprehensive cybersecurity platform designed to protect users' digital identities by monitoring data breaches, providing security recommendations, managing passwords, and enforcing security best practices.\n\n## 🎨 Design Theme\n\n### Color Palette (No Glass Morphism)\n- **Primary Navy Blue**: #001F3F\n- **Secondary White**: #FFFFFF\n- **Accent Light Blue**: #0074D9\n- **Dark Slate**: #1A3A52\n- **Light Gray**: #F5F5F5\n\n### Design Principles\n- Solid, professional appearance\n- High contrast for accessibility\n- Clean, minimalist UI\n- No transparency/glass effects\n- Clear hierarchy and readability\n\n---\n\n## 🏗️ System Architecture\n\n### Frontend Architecture\n\n```\nFrontend (React.js)\n├── Pages\n│   ├── Dashboard - Security Overview\n│   ├── Breach Monitoring - Track compromised data\n│   ├── Password Manager - Secure storage & management\n│   ├── Security Settings - User preferences & 2FA\n│   ├── Alerts - Breach notifications\n│   └── Recommendations - Security suggestions\n├── Components\n│   ├── Navbar - Navigation\n│   ├── PrivateRoute - Access control\n│   ├── Toast - Notifications\n│   ├── Cards - UI components\n│   └── Forms - User input\n├── Redux Store\n│   ├── auth - Authentication state\n│   ├── breaches - Breach data\n│   ├── passwords - Password management\n│   ├── alerts - Notification state\n│   └── recommendations - Security tips\n└── Services\n    └── API - Backend communication\n```\n\n### Backend Architecture\n\n```\nBackend (Node.js/Express)\n├── Routes\n│   ├── /api/auth - Authentication endpoints\n│   ├── /api/users - User management\n│   ├── /api/breach - Breach detection & alerts\n│   ├── /api/passwords - Password management\n│   ├── /api/alerts - Alert notifications\n│   ├── /api/recommendations - Security suggestions\n│   └── /api/dashboard - Analytics & overview\n├── Models\n│   ├── User - User data & security settings\n│   ├── BreachAlert - Breach notifications\n│   ├── StoredPassword - Encrypted passwords\n│   ├── Recommendation - Security suggestions\n│   └── AuditLog - Security audit trail\n├── Services\n│   ├── BreachDetectionService - Monitor breaches\n│   ├── EncryptionService - Data encryption/decryption\n│   ├── NotificationService - Email/SMS alerts\n│   ├── PasswordAnalysisService - Strength checking\n│   └── RecommendationEngine - Security analysis\n├── Middleware\n│   ├── Authentication - JWT validation\n│   ├── Authorization - Role-based access\n│   ├── Validation - Input validation\n│   ├── ErrorHandling - Error management\n│   └── Logging - Request logging\n└── Config\n    ├── database.js - PostgreSQL connection\n    ├── redis.js - Redis cache connection\n    └── logger.js - Winston logging\n```\n\n### Database Schema\n\n#### Users Table\n```sql\nColumns:\n- id (UUID) - Primary key\n- email (VARCHAR) - User email (unique)\n- username (VARCHAR) - Username (unique)\n- password_hash (VARCHAR) - Bcrypt hashed password\n- full_name (VARCHAR) - User's full name\n- phone_number (VARCHAR) - Contact number\n- verified_email (BOOLEAN) - Email verification status\n- two_factor_enabled (BOOLEAN) - 2FA status\n- two_factor_method (VARCHAR) - 2FA method (SMS/Email/Auth App)\n- notification_preferences (JSONB) - User notification settings\n- last_login (TIMESTAMP) - Last login timestamp\n- created_at (TIMESTAMP) - Account creation date\n- updated_at (TIMESTAMP) - Last update date\n\nIndexes:\n- idx_users_email\n- idx_users_username\n```\n\n#### Breach Alerts Table\n```sql\nColumns:\n- id (UUID) - Primary key\n- user_id (UUID) - Foreign key to users\n- breach_name (VARCHAR) - Name of the breach\n- data_exposed (TEXT) - Types of data exposed\n- exposure_date (DATE) - When the breach occurred\n- compromised_data (JSONB) - Details of compromised data\n- severity_level (VARCHAR) - HIGH/MEDIUM/LOW\n- is_read (BOOLEAN) - Alert read status\n- created_at (TIMESTAMP) - Alert creation date\n- updated_at (TIMESTAMP) - Last update date\n\nIndexes:\n- idx_breach_alerts_user_id\n```\n\n#### Stored Passwords Table\n```sql\nColumns:\n- id (UUID) - Primary key\n- user_id (UUID) - Foreign key to users\n- service_name (VARCHAR) - Service/website name\n- username (VARCHAR) - Account username\n- email (VARCHAR) - Account email\n- encrypted_password (TEXT) - AES-256 encrypted password\n- password_strength (VARCHAR) - STRONG/MEDIUM/WEAK\n- last_changed (TIMESTAMP) - Last password change\n- created_at (TIMESTAMP) - Record creation date\n- updated_at (TIMESTAMP) - Last update date\n\nIndexes:\n- idx_stored_passwords_user_id\n```\n\n#### Recommendations Table\n```sql\nColumns:\n- id (UUID) - Primary key\n- user_id (UUID) - Foreign key to users\n- recommendation_type (VARCHAR) - Type of recommendation\n- title (VARCHAR) - Recommendation title\n- description (TEXT) - Detailed description\n- priority (VARCHAR) - CRITICAL/HIGH/MEDIUM/LOW\n- status (VARCHAR) - NEW/ACKNOWLEDGED/RESOLVED\n- created_at (TIMESTAMP) - Creation date\n- updated_at (TIMESTAMP) - Last update date\n\nIndexes:\n- idx_recommendations_user_id\n```\n\n---\n\n## 🔐 Security Features\n\n### 1. Authentication & Authorization\n- JWT-based authentication\n- Bcrypt password hashing (10 rounds)\n- Refresh token mechanism\n- Session management\n- Role-based access control (RBAC)\n\n### 2. Data Encryption\n- AES-256 encryption for sensitive data\n- Encrypted password storage\n- Secure key management\n- Data in transit: HTTPS/TLS\n- Data at rest: Encrypted columns\n\n### 3. Security Headers\n- Helmet.js for security headers\n- CORS protection\n- Rate limiting (100 requests per 15 minutes)\n- CSRF protection\n- XSS prevention\n\n### 4. Breach Monitoring\n- Real-time breach database integration\n- Pattern matching algorithms\n- Automated alerts when user data detected\n- Email/SMS notifications\n- Alert prioritization by severity\n\n### 5. Password Management\n- Secure encrypted storage\n- Password strength analysis\n- Auto-generation with customizable options\n- Change history tracking\n- Compromised password detection\n\n---\n\n## 📡 API Endpoints\n\n### Authentication\n```\nPOST   /api/auth/register      - User registration\nPOST   /api/auth/login         - User login\nPOST   /api/auth/logout        - User logout\nPOST   /api/auth/refresh-token - Refresh JWT token\nPOST   /api/auth/forgot-password - Request password reset\nPOST   /api/auth/reset-password - Reset password with token\nPOST   /api/auth/verify-email  - Verify email address\nGET    /api/auth/verify-token  - Verify JWT token validity\n```\n\n### Users\n```\nGET    /api/users/profile      - Get user profile\nPUT    /api/users/profile      - Update user profile\nGET    /api/users/security     - Get security settings\nPUT    /api/users/security     - Update security settings\nGET    /api/users/score        - Get security score\n```\n\n### Breach Monitoring\n```\nGET    /api/breach/alerts      - Get all breach alerts\nGET    /api/breach/alerts/:id  - Get specific alert\nPOST   /api/breach/scan        - Scan user data against breaches\nPUT    /api/breach/alerts/:id  - Mark alert as read\nDELETE /api/breach/alerts/:id  - Delete alert\n```\n\n### Password Manager\n```\nGET    /api/passwords          - List all stored passwords\nPOST   /api/passwords          - Add new password\nGET    /api/passwords/:id      - Get password details\nPUT    /api/passwords/:id      - Update password\nDELETE /api/passwords/:id      - Delete password\nPOST   /api/passwords/generate - Generate strong password\nPOST   /api/passwords/analyze  - Analyze password strength\n```\n\n### Alerts\n```\nGET    /api/alerts             - Get all alerts\nGET    /api/alerts/unread      - Get unread alerts count\nPUT    /api/alerts/:id/read    - Mark alert as read\nPOST   /api/alerts/subscribe   - Subscribe to alerts\n```\n\n### Recommendations\n```\nGET    /api/recommendations    - Get security recommendations\nPOST   /api/recommendations/acknowledge - Acknowledge recommendation\nPUT    /api/recommendations/:id - Update recommendation status\n```\n\n### Dashboard\n```\nGET    /api/dashboard/overview - Get dashboard overview\nGET    /api/dashboard/stats    - Get security statistics\nGET    /api/dashboard/recent   - Get recent activities\n```\n\n---\n\n## 🚀 Deployment\n\n### Docker Compose Services\n\n1. **PostgreSQL** (postgres:15-alpine)\n   - Primary database\n   - Data persistence with volumes\n   - Health checks enabled\n\n2. **Redis** (redis:7-alpine)\n   - Caching layer\n   - Session storage\n   - Real-time notifications\n\n3. **Backend** (Node.js)\n   - Express.js API server\n   - Port 5000\n   - Auto-restart policy\n\n4. **Frontend** (React.js)\n   - React development/production server\n   - Port 3000\n   - Auto-restart policy\n\n### Environment Configuration\n\nRequired environment variables (see .env.example):\n- Database credentials\n- JWT secrets\n- API keys for third-party services\n- Encryption keys\n- Redis configuration\n- CORS settings\n\n---\n\n## 📊 Technology Stack\n\n### Frontend\n- **React.js** 18.2.0 -
